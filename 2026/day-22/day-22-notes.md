# Day 22 Notes – Introduction to Git

## 1. Difference between git add and git commit

git add moves changes to the staging area.
git commit saves staged changes permanently in the repository.

## 2. What is the staging area?

The staging area is an intermediate layer where we select changes before committing them.

## 3. What does git log show?

git log shows commit history including commit ID, author, date, and message.

## 4. What is the .git folder?

.git is a hidden directory that stores the complete Git repository history.

If deleted, the repository history will be lost.

## 5. Working Directory vs Staging Area vs Repository

Working Directory → where files are edited  
Staging Area → where selected changes are prepared  
Repository → where commits are stored permanently
