# Day 23 – Git Branching & GitHub

## 1. What is a branch in Git?

A branch is an independent line of development. It allows developers to work on features or fixes without affecting the main code.

## 2. Why use branches instead of committing to main?

Branches isolate changes so unstable code does not break the main production branch.

## 3. What is HEAD in Git?

HEAD is a pointer that refers to the current commit and branch you are working on.

## 4. What happens when switching branches?

Git updates the working directory to match the snapshot of the branch you switch to.

## Difference between origin and upstream

origin → your forked repository  
upstream → original repository

## Difference between git fetch and git pull

git fetch downloads remote changes but does not merge them.  
git pull downloads and merges changes automatically.

## Clone vs Fork

Clone → copy repository locally.  
Fork → create your own copy of repository on GitHub.

Fork is used when contributing to someone else's repository.
