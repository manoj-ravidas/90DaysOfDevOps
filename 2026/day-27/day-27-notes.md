# Day 27 – GitHub Profile Makeover

## Objective

Improve my GitHub profile so that recruiters and hiring managers can clearly understand my skills and projects.

---

# Profile Audit

Before making changes I reviewed my GitHub profile from a recruiter's perspective.

Questions I considered:

* Is my profile picture professional?
* Does my bio describe what I do?
* Are my repositories organized?
* Do my repositories have clear descriptions and README files?

---

# Changes I Made

## 1. Created a GitHub Profile README

I created a repository with the same name as my GitHub username.

Example:

username/username

This allows the README to appear on my GitHub profile page.

The README includes:

* A short introduction
* My DevOps learning journey
* Tools and technologies I am learning
* Links to important repositories
* Contact information

---

## 2. Organized My Repositories

I organized my work into structured repositories:

### 90DaysOfDevOps

Contains daily learning tasks and submissions.

### shell-scripts

Contains all shell scripting practice and automation scripts.

### python-scripts

Contains Python automation scripts and practice projects.

### devops-notes

Contains notes, cheat sheets, and DevOps learning references.

---

## 3. Added README Files

Each repository now includes:

* A clear description
* A README explaining the project
* Organized folder structure

---

## 4. Pinned Important Repositories

I pinned my best repositories on my GitHub profile so visitors can quickly see my most relevant work.

Example pinned repositories:

* 90DaysOfDevOps
* shell-scripts
* python-scripts
* devops-notes
* github-actions-practice
